## 1. 앱이랑 CLI가 뭐가 다른 거야?

Claude를 쓰는 방법은 두 가지예요. **둘 다 같은 AI**예요.

| | 앱 (데스크톱) | CLI (터미널) |
|---|---|---|
| 조작 | 클릭, 드래그 | 타이핑, 명령어 |
| 난이도 | 쉬움 | 익숙해지면 빠름 |
| 기능 | 충분 | 더 많음 |

> 💡 비유: 같은 차인데 오토(앱)와 스틱(CLI)의 차이예요. 목적지는 같은데 조작 방식만 달라요.

<details>
<summary>▸ 앱(데스크톱)은 이런 분에게 맞아요</summary>

- 컴퓨터에서 검은 화면(터미널)을 써본 적 없는 분
- 파일을 드래그해서 넣고, 클릭으로 조작하고 싶은 분
- "일단 바로 써보고 싶다"는 분

대부분의 초심자는 앱으로 시작하는 걸 권해요!

</details>

<details>
<summary>▸ CLI(터미널)는 이런 분에게 맞아요</summary>

- 터미널이 뭔지 알고, 타이핑이 빠른 분
- 자동화나 스크립트를 돌려보고 싶은 분
- "더 세밀하게 컨트롤하고 싶다"는 분

앱보다 기능이 더 많고, 속도가 빨라요.

</details>

## 2. 폴더 = AI의 작업 공간

Claude Code를 열면 "어떤 폴더에서 작업할 건가?"를 정해요.
**AI가 볼 수 있는 범위가 이 폴더로 정해져요.**

> 💡 비유: 사무실에서 서랍을 하나 열어서 AI 앞에 놓는 거예요. AI는 그 서랍 안의 서류만 봐요.

---

## 3. CLAUDE.md는 두 종류가 있어요

어디에 넣느냐에 따라 역할이 달라져요. 어머, 이거 모르시는 분 많더라고요!

| | Global CLAUDE.md | Project CLAUDE.md |
|---|---|---|
| **위치** | ~/.claude/CLAUDE.md | 프로젝트폴더/CLAUDE.md |
| **적용 범위** | 어떤 폴더를 열든 항상 | 이 폴더에서만 |
| **역할** | 🪪 명함 | 📋 브리핑 메모 |

<details>
<summary>▸ 🪪 Global CLAUDE.md — 모든 프로젝트에 적용</summary>

위치: `~/.claude/CLAUDE.md`
어떤 폴더를 열든 AI가 항상 읽는 파일이에요.

| 넣을 것 | 예시 |
|---|---|
| 나는 누구인가 | 변호사 / 10년차 세무사 |
| 항상 지킬 규칙 | 한국어로 답해, 결론부터 써 |
| 공통 말투 | 격식체, 반말 금지 |

> 💡 비유: 명함이에요. 누구를 만나든 건네는 것.

</details>

<details>
<summary>▸ 📋 Project CLAUDE.md — 이 폴더에서만 적용</summary>

위치: `소송-홍길동/CLAUDE.md`
이 폴더를 열었을 때만 AI가 읽어요. Global 위에 추가로 적용돼요.

| 넣을 것 | 예시 |
|---|---|
| 프로젝트 맥락 | 부당해고 사건, 을 입장 |
| 이 작업의 규칙 | 근로기준법 기준으로 검토 |
| 참고할 파일 | 계약서.pdf, 답변서.docx |

> 💡 비유: 프로젝트 브리핑 메모. 이 건에 대한 안내.

</details>

<details>
<summary>▸ 두 개가 동시에 작동하는 원리</summary>

AI가 폴더를 열면 이 순서로 읽어요:

| 순서 | 읽는 파일 | AI가 이해하는 것 |
|---|---|---|
| 1 | Global CLAUDE.md | "이 사람은 변호사, 한국어, 결론부터" |
| 2 | Project CLAUDE.md | "이 건은 부당해고, 을 입장, 근로기준법" |

> 💡 병원 비유: Global = 진료 지침서 (모든 환자에 적용), Project = 환자 차트 (이 환자에만 적용)

</details>

---

## 4. 세션 = 전화 한 통

세션 = 하나의 대화 단위예요. 대화를 종료하고 새로 시작하면, 새 세션이에요.

| 상황 | 이전 대화 | CLAUDE.md |
|---|---|---|
| 같은 세션 안 | ✅ 기억함 | ✅ 읽음 |
| 새 세션 시작 | ❌ 모름 | ✅ 매번 읽음 |

> 💡 비유: 전화 통화예요. 통화 중에는 기억하지만, 끊고 다시 걸면 처음부터. CLAUDE.md는 매번 읽으니까 — **잊으면 안 되는 건 거기에 적어두세요.**

---

## 5. 컨텍스트 윈도우 = 책상 크기

컨텍스트 윈도우 = AI가 한 번에 기억할 수 있는 양이에요.
전화 통화를 한 시간 넘게 하면 처음 얘기가 가물가물한 것과 같아요.

> 💡 비유: 책상 크기예요. 서류를 계속 올리면 맨 밑의 서류가 밀려 떨어져요. CLAUDE.md는 책상에 고정한 클리어파일 — 항상 보여요.

<details>
<summary>▸ 모델별 컨텍스트 크기</summary>

| 모델 | 컨텍스트 윈도우 | 비유 |
|---|---|---|
| Claude Sonnet | ~200K 토큰 | A4 300~400장 |
| Claude Opus | ~1M 토큰 | A4 ~1,500장 |

대부분의 업무 대화에서는 넘칠 일이 거의 없어요.

</details>

<details>
<summary>▸ 주의할 상황 & 해결법</summary>

| 이런 경우 주의 | 해결법 |
|---|---|
| 100페이지+ PDF 통째로 넣을 때 | 필요한 부분만 발췌해서 넣기 |
| 한 세션에서 몇 시간 쉬지 않고 대화 | 새 세션 시작 (전화 끊고 다시 걸기) |
| 파일 여러 개 동시에 넣을 때 | 한 세션 한 업무 원칙 |

> 핵심 해결법: 잊으면 안 되는 건 CLAUDE.md에 적어두세요. 세션이 바뀌어도 매번 읽어요.

</details>

## 전체 구조 한눈에

```
나(마법사)
  └── Claude Code (지팡이)
        ├── 앱 모드 (오토) 또는 CLI 모드 (스틱)
        │
        ├── Global CLAUDE.md (명함 — 항상 읽음)
        │
        └── 폴더 (서랍)
              ├── Project CLAUDE.md (브리핑 메모)
              ├── 내 파일들
              └── 세션 (전화 한 통)
                    └── 컨텍스트 윈도우 (책상 크기)
```
