# DA 수업 #7 — It's leviOsa! 내 말투로 쓰게 만드는 법 🪶

> AI가 쓴 글을 읽으면 "AI가 썼네" 딱 티 나죠?
> 그건 AI가 못 써서가 아니라, **내 말투를 모르기 때문**이에요. 오늘 가르쳐줄 거예요!

---

## 문제: AI 기본 톤 = 다 똑같다

| | AI 기본 톤 | 내가 원하는 톤 |
|---|---|---|
| 인사말 | "안녕하세요, ~에 대해 알려드리겠습니다" | "결론부터 말씀드리면" |
| 문체 | 교과서 같은 설명체 | 실무에서 쓰는 간결체 |
| 길이 | 길고 친절 | 핵심만 |
| 느낌 | 콜센터 직원 | 10년 차 동료 |

> 💡 비유: AI의 기본 발음은 "레비오사~"예요. 내 발음 "레비**오**사!"로 교정해줘야 해요. It's levi**O**sa, not levio**SA**! 🪶

---

## 방법 1: 내 글 샘플 보여주기 (가장 빠름)

내가 쓴 글을 AI에게 보여주고 "이 톤으로 써줘"라고 하면 돼요.

| 단계 | 할 일 |
|---|---|
| 1 | 내가 잘 썼다고 생각하는 글 1~2개를 준비 |
| 2 | AI에게 붙여넣기 |
| 3 | "이 글의 말투와 구조를 분석해줘" |
| 4 | "앞으로 이 톤으로 써줘" |

```
[내 글 샘플 붙여넣기]

위 글의 말투, 문장 길이, 구조를 분석하고,
앞으로 이 스타일로 글을 써줘.
```

> 💡 비유: 펜시브(기억 저장소, Pensieve)에 내 기억을 넣어주는 거예요. AI가 내 글쓰기 습관을 직접 보고 배우는 거죠.

---

## 방법 2: 규칙으로 지정하기 (정밀 조정)

샘플 없이도, 규칙을 말로 알려줄 수 있어요.

| 규칙 항목 | 예시 |
|---|---|
| 문체 | "격식체, ~합니다" |
| 길이 | "한 문단 3줄 이내" |
| 구조 | "결론 → 근거 → 다음 행동 순서" |
| 금지어 | "~인 것 같습니다" 사용 금지 |
| 톤 | "친절하되 군더더기 없이" |

```
글쓰기 규칙:
- 격식체 (~합니다)
- 결론부터 쓰고, 근거는 뒤에
- 한 문단 3줄 이내
- "~인 것 같습니다" 금지
- 전문 용어는 첫 등장 시 한 번만 설명
```

---

## 방법 3: CLAUDE.md에 저장하기 (영구 적용)

매번 말하기 귀찮잖아요? **마법서(CLAUDE.md)에 적어두면 매 세션마다 자동 적용**돼요.

| | 매번 지시 | CLAUDE.md에 저장 |
|---|---|---|
| **효과 범위** | 이 세션만 | 모든 세션 |
| **수고** | 매번 복붙 | 한 번만 |
| **비유** | 매번 주문 외우기 | 마법서에 적어두기 |

<details>
<summary>▸ CLAUDE.md에 넣을 말투 규칙 템플릿 (복붙용)</summary>

```markdown
## 글쓰기 규칙

### 문체
- 격식체 (~합니다)
- 결론부터, 근거는 뒤에
- 한 문단 3줄 이내

### 금지
- "~인 것 같습니다" — "~입니다"로 대체
- 불필요한 인사말/마무리 생략
- "도움이 되셨으면 좋겠습니다" 금지

### 구조
- 표 > 글 (비교는 반드시 표)
- 번호 목록 > 글머리 기호
- 3줄 이상이면 소제목 붙이기
```

</details>

## Before / After 비교

| 항목 | Before (기본 톤) | After (내 톤 적용) |
|---|---|---|
| 이메일 | "안녕하세요, 지난 미팅에서 논의된 사항에 대해 말씀드리고자 이메일 드립니다..." | "지난 미팅 후속입니다. 결론: A안으로 진행하겠습니다." |
| 보고서 서두 | "본 보고서는 ~에 대한 분석을 목적으로 작성되었으며..." | "핵심: 전년 대비 매출 12% 증가. 원인은 3가지." |
| 환자 안내문 | "해당 시술에 대해 아래와 같이 안내드리오니 참고하시기 바랍니다" | "이 시술은 ~입니다. 시간은 30분, 주의사항은 2가지예요." |

## 전문가별 말투 예시

<details>
<summary>▸ 변호사 — 법률 문서 톤</summary>

| 규칙 | 설명 |
|---|---|
| 문체 | 격식체, 법률 용어 사용 |
| 구조 | 쟁점 → 관련 법조문 → 검토 의견 |
| 주의 | "~로 사료됩니다"는 법원 문서에서만, 내부 메모에선 "~입니다" |

```
## 글쓰기 규칙 (법률 문서)
- 격식체, "~하는 바입니다" 톤
- 쟁점 번호를 매겨서 구조화
- 관련 법조문은 반드시 인용
- 내부 검토 메모는 결론부터
```

</details>

<details>
<summary>▸ 의사 — 진료/학술 톤</summary>

| 규칙 | 설명 |
|---|---|
| 환자 안내문 | 전문 용어 없이, 중학생이 읽어도 이해되는 수준 |
| 학술 문서 | 영문 의학 용어 병기, IMRAD 구조 |
| 내부 메모 | 약어 허용, 핵심만 |

```
## 글쓰기 규칙 (진료)
- 환자용: 쉬운 말, 전문 용어 뒤에 (쉬운 설명) 병기
- 학술용: 영문 용어 우선, 한글 병기
- 내부: 약어 OK, 3줄 이내
```

</details>

<details>
<summary>▸ 회계사/세무사 — 보고서 톤</summary>

| 규칙 | 설명 |
|---|---|
| 고객 보고서 | 결론(숫자) → 근거(표) → 제언 |
| 내부 검토 | 이상치 중심, 표 우선 |
| 세무 안내 | 법조문 번호 포함, 기한 강조 |

```
## 글쓰기 규칙 (회계/세무)
- 숫자는 반드시 표에 넣기
- 결론(핵심 수치) → 근거(비교 표) → 다음 행동
- 기한이 있는 항목은 볼드 처리
```

</details>

<details>
<summary>▸ 약사 — 복약/상담 톤</summary>

| 규칙 | 설명 |
|---|---|
| 복약 안내 | 환자 눈높이, 복용 시간 강조, 이모지 허용 |
| 약물 상호작용 | 위험도 등급 필수, 근거 병기 |
| 내부 메모 | 약품명(성분명) 형식 |

```
## 글쓰기 규칙 (약사)
- 복약 안내: 하루 일과 기준으로 복용 시간 설명
- 전문 용어 뒤에 (쉬운 설명) 병기
- 위험 정보는 반드시 위험도 등급 표시
```

</details>
