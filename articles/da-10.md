# DA 수업 #10 — 발표자료, 반나절이 30분으로 🎤

> 발표자료 만들 때 가장 오래 걸리는 건 뭘까요?
> 디자인? 아니에요. **"뭘 넣을지" 고민하는 시간**이에요. AI한테 구조부터 시키면 반나절이 30분으로 줄어요!

---

## 핵심 비유

> 💡 머글(일반인)은 빈 파치먼트(빈 슬라이드)를 앞에 놓고 고민해요. 마법사(AI 사용자)는 주문을 외우면 골격이 먼저 나타나고, 살을 붙이고, 마지막에 옷을 입혀요. 구조(Outline) → 내용(Content) → 디자인(Design) 순서!

---

## 기존 과정 vs AI 과정

| 단계 | 기존 (머글 방식) | AI 활용 (마법사 방식) |
|---|---|---|
| 1. 구조잡기 | 빈 화면에서 목차 고민 (1~2시간) | "10장짜리 발표 목차 잡아줘" (2분) |
| 2. 내용 채우기 | 한 장씩 내용 작성 (2~3시간) | "각 슬라이드 내용을 채워줘" (5분) |
| 3. 디자인 | 정렬, 폰트, 색상 조정 (1~2시간) | "각 슬라이드에 맞는 레이아웃 제안해줘" (3분) |
| 4. 검토/수정 | 동료에게 보여주고 수정 (반복) | "발표자 관점에서 빠진 내용 있는지 확인해줘" (2분) |
| **합계** | **반나절~하루** | **30분~1시간** |

> 💡 비유: 빈 파치먼트(빈 슬라이드)에 글씨를 하나씩 쓰는 대신, 마법 깃펜(AI)이 구조를 먼저 그려주고 나는 다듬기만 해요.

---

## 발표자료 주문 — 실전 예시

### 10장짜리 발표를 AI로 만드는 과정

**Step 1: 구조잡기**

```
[주제: 우리 의원의 2025년 성과와 2026년 계획]

10장짜리 발표자료의 목차를 잡아줘.
- 청중: 의원 내부 직원
- 발표 시간: 15분
- 핵심 메시지: 올해 성장했고, 내년에 더 성장할 것
- 각 슬라이드의 제목과 핵심 메시지를 표로
```

**Step 2: 내용 채우기**

```
위 목차 기준으로 각 슬라이드의 내용을 채워줘.
- 슬라이드당 핵심 메시지 1개
- 글머리 3~4개 이내
- 숫자가 있는 슬라이드는 차트 유형도 추천해줘
```

**Step 3: 디자인 제안**

```
각 슬라이드에 적합한 레이아웃을 추천해줘.
- 제목만 있는 슬라이드: 큰 제목 + 부제
- 데이터 슬라이드: 차트 위치 + 핵심 수치 강조
- 비교 슬라이드: 2단 레이아웃
```

**Step 4: 발표 대본**

```
각 슬라이드에 대한 발표 대본을 써줘.
- 슬라이드당 30초~1분 분량
- 자연스러운 구어체
- 슬라이드 전환 멘트 포함
```

## 좋은 발표자료의 원칙

| 원칙 | 설명 | AI에게 시키는 법 |
|---|---|---|
| 1장 1메시지 | 한 슬라이드에 메시지 하나 | "슬라이드당 핵심 메시지를 하나로 줄여줘" |
| 글은 적게, 그림은 많이 | 글머리 3~4개 이내 | "텍스트를 줄이고 차트/도식으로 바꿔줘" |
| 숫자는 차트로 | 표보다 차트가 빠르게 읽힘 | "이 표를 차트로 바꿔줘" (DA#9) |
| 스토리 구조 | 문제 → 원인 → 해결 → 기대효과 | "문제-원인-해결-기대효과 구조로 재배치해줘" |
| 청중 맞춤 | 같은 내용도 청중에 따라 다르게 | "청중이 [누구]인 점을 감안해서 톤 조정해줘" |

## 발표자료 만들기 전체 흐름

```
1. 구조잡기 — "~장짜리 목차 잡아줘"
2. 내용 채우기 — "각 슬라이드 내용을 채워줘"
3. 시각화 — "숫자는 차트로, 비교는 표로" (DA#8, #9)
4. 디자인 제안 — "레이아웃 추천해줘"
5. 발표 대본 — "슬라이드별 대본 써줘"
6. 검토 — "빠진 내용 없는지, 흐름이 자연스러운지 확인해줘"
```

> 정리: **구조 → 내용 → 디자인 → 대본.** 이 순서로 AI에게 시키면 발표자료가 30분 안에 나와요. 빈 화면 앞에서 고민하는 시간이 가장 비싼 비용이에요.
