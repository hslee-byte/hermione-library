# DA 수업 #11 — 지팡이 코어 선택법 🪄

> 올리밴더 지팡이 가게에서 지팡이를 고를 때,
> 가장 중요한 건 **코어(Core)**예요. 어떤 코어를 쓰느냐에 따라 마법의 성격이 달라지거든요.
> Claude에도 세 가지 코어(모델)가 있어요.

---

## 세 가지 코어 — 한눈에 비교

| | 🔥 불사조 깃털 (Opus) | 🦄 유니콘 꼬리 (Sonnet) | 🐉 용의 심장줄 (Haiku) |
|---|---|---|---|
| **한마디** | 가장 깊고 정교한 마법 | 균형 잡힌 만능형 | 빠르고 가벼운 속사 주문 |
| **깊이** | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| **속도** | 느림 (깊이 생각) | 중간 | 매우 빠름 |
| **비용** | 비쌈 ($15/100만 토큰) | 중간 ($3/100만 토큰) | 저렴 ($1/100만 토큰) |
| **맥락 길이** | 200K | 200K | 200K |
| **언제 쓰나** | 복잡한 판단, 전략, 긴 문서 분석 | 대부분의 업무 | 단순 반복, 분류, 짧은 요약 |

> 💡 비유: 올리밴더가 말했죠 — "지팡이가 마법사를 선택한다."
> 하지만 머글 세계에서는 **업무가 코어(모델)를 선택해요.** 🪄

---

## 90%는 Sonnet이면 충분해요

이게 오늘 수업의 핵심이에요.

| 오해 | 실제 |
|---|---|
| "비싼 게 항상 좋겠지" | Opus는 깊이가 필요한 10%의 작업에만 차이를 만들어요 |
| "빠른 게 좋겠지" | Haiku는 깊이가 필요한 작업에서는 아쉬워요 |
| "잘 모르겠으니 Opus" | 비용만 올라가고, 체감 차이는 거의 없는 경우가 대부분 |

> 💡 비유: 편의점 갈 때 포르쉐를 몰 필요 없어요.
> 대부분의 업무는 소나타(Sonnet)면 충분해요 🚗

---

## 상황별 추천 코어

| 상황 | 추천 모델 | 이유 |
|---|---|---|
| 계약서 위험 조항 검토 | 🦄 Sonnet | 구조화된 분석이면 충분 |
| M&A 계약서 전략적 분석 | 🔥 Opus | 맥락 간 연결, 숨겨진 리스크 판단 |
| 진료 기록 요약 | 🦄 Sonnet | 패턴화된 정리 작업 |
| 복잡한 감별진단 논의 | 🔥 Opus | 다중 변수 추론 |
| 이메일 초안 | 🦄 Sonnet | 충분 |
| 100건 서류 분류 | 🐉 Haiku | 빠르고 저렴하게 반복 |
| 세금 신고 체크리스트 | 🦄 Sonnet | 충분 |
| 약물 상호작용 검토 | 🦄 Sonnet | 구조화된 비교 |
| 전략 보고서 초안 | 🔥 Opus | 깊이 있는 논증 구조 |
| 간단한 번역/요약 | 🐉 Haiku | 속도와 비용 우선 |

---

## Opus를 꺼내야 할 때 — 체크리스트

아래 중 **2개 이상**이면 Opus를 쓰세요:

| # | 조건 | 체크 |
|---|---|---|
| 1 | 문서가 50페이지 이상이고, 전체 맥락을 연결해야 한다 | ☐ |
| 2 | "이건 왜 이런 거지?" — 판단/추론이 핵심이다 | ☐ |
| 3 | 여러 문서를 동시에 비교해야 한다 | ☐ |
| 4 | 틀렸을 때 리스크가 크다 (법률, 의료, 재무) | ☐ |
| 5 | 전략적 의사결정의 근거를 만들고 있다 | ☐ |

> 💡 0~1개? Sonnet. 2개 이상? Opus 소환!

---

## Haiku가 빛나는 순간

| 작업 유형 | 예시 |
|---|---|
| 대량 분류 | 100건 민원을 유형별로 분류 |
| 단순 변환 | 텍스트 → 표, JSON → 요약 |
| 예/아니오 판단 | "이 문장이 긍정인가 부정인가" |
| 빠른 초안 | 1줄 답변, 제목 후보 생성 |

> 💡 비유: 용의 심장줄 지팡이는 화력은 세지만 단순해요.
> 반복 작업을 빠르게 쏘아대는 데 딱이에요 🐉

---

<details>
<summary>▸ 모델 바꾸는 법 (Claude Code에서)</summary>

**방법 1: 대화 중 바꾸기**

```
/model opus
```
또는
```
/model haiku
```

대화 중간에 입력하면 그 시점부터 모델이 바뀌어요.

**방법 2: 시작할 때 지정하기**

```bash
claude --model opus
claude --model sonnet
claude --model haiku
```

**방법 3: CLAUDE.md에 기본값 적기**

```markdown
# 모델 설정
기본 모델: sonnet
복잡한 분석 요청 시: opus 사용 권장
```

> 팁: 대부분은 기본(Sonnet)으로 두고, 복잡한 작업일 때만 `/model opus`로 전환하세요.

</details>

<details>
<summary>▸ API 사용 시 모델명</summary>

| 🦄 Sonnet | `claude-sonnet-4-20250514` |
| 🐉 Haiku | `claude-haiku-3-5-20241022` |

> 개발자가 아니라면 이 부분은 넘어가셔도 돼요!

</details>

## 비용 감각 — 실제로 얼마나 차이 날까

| 작업 | Opus 예상 비용 | Sonnet 예상 비용 | Haiku 예상 비용 |
|---|---|---|---|
| 계약서 10페이지 분석 | ~$0.30 | ~$0.06 | ~$0.02 |
| 100건 서류 분류 | ~$3.00 | ~$0.60 | ~$0.20 |
| 한 달 일반 사용 | ~$150+ | ~$30 | ~$10 |

> 💡 Pro 플랜($20/월)이면 Sonnet 무제한, Opus는 제한적 사용.
> 대부분의 전문가에게 Pro 플랜 + Sonnet 기본이 최적이에요.
