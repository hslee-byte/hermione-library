# DA 수업 #13 — 나만의 주문 만들기 ⚡

> 매번 "너는 ~야, 이 형식으로 ~해줘, 주의사항은 ~"을 반복하고 계시죠?
> 그걸 **주문서(스킬, Custom Slash Command)**에 적어두면,
> `/계약서검토`처럼 한 마디로 실행돼요.

---

## 매번 외우기 vs 주문서에 적어두기

| | 📢 매번 타이핑 | 📜 스킬(주문서) |
|---|---|---|
| **시작 시간** | 주문 문구 작성에 1~2분 | `/스킬이름` 한 줄, 1초 |
| **일관성** | 매번 조금씩 달라짐 | 항상 같은 품질 |
| **실수** | 규칙 빠뜨리기 쉬움 | 규칙이 주문서에 내장 |
| **공유** | 복사-붙여넣기 | 팀원에게 파일 하나 공유 |
| **비유** | 매번 주문을 기억에서 외우기 | 주문서에 적어두고 읽기 ⚡ |

> 💡 비유: 해리가 매번 "엑스펙토 패트로눔"을 새로 연습하는 것 vs
> 주문서에 적어두고 바로 시전하는 것. 결과는 같지만 속도가 다르죠.

---

## 스킬 만드는 법 — 3단계

| 단계 | 할 일 | 설명 |
|---|---|---|
| **1단계** | 폴더 만들기 | `.claude/commands/` 폴더 안에 파일 생성 |
| **2단계** | 주문서 작성 | 마크다운 파일에 주문 내용 적기 |
| **3단계** | 실행 | `/스킬이름`으로 호출 |

---

### 1단계: 폴더 확인

```
프로젝트 폴더/
└── .claude/
    └── commands/          ← 여기에 스킬 파일을 넣어요
        ├── 계약서검토.md
        ├── 환자안내문.md
        └── 매출분석.md
```

> 💡 `.claude/commands/`는 프로젝트별 스킬 저장소예요.
> `~/.claude/commands/`에 넣으면 어디서든 쓸 수 있는 글로벌 스킬이 돼요.

---

### 2단계: 주문서 작성 — 예시

**파일명:** `.claude/commands/계약서검토.md`

```markdown
이 계약서를 검토해줘.

## 역할
너는 10년차 기업법 변호사야.

## 검토 기준
1. 을에게 불리한 조항을 우선 검토
2. 손해배상 한도, 해지 조건, 지재권 귀속에 집중
3. 리스크 등급을 상/중/하로 분류

## 출력 형식
| 조항 번호 | 내용 요약 | 리스크 등급 | 이유 | 수정 제안 |

## 주의
- 확정 판단이 아닌 검토 의견임을 명시해줘
```

> 이 파일을 저장하면, `/계약서검토`로 실행할 수 있어요!

---

### 3단계: 실행

```
> /계약서검토
```

이 한 줄이면 위의 긴 주문이 자동 실행돼요.

> 💡 `$ARGUMENTS`를 넣으면 실행할 때 추가 입력을 받을 수 있어요.
> 예: "이 계약서를 검토해줘: $ARGUMENTS" → `/계약서검토 첨부된 NDA 계약서`

## 스킬 vs CLAUDE.md — 뭐가 다른가요?

| | CLAUDE.md (마법서) | 스킬 (주문서) |
|---|---|---|
| **언제 작동** | 항상 (폴더 열면 자동) | 호출할 때만 (`/이름`) |
| **용도** | 배경 설정, 기본 규칙 | 특정 작업 실행 |
| **비유** | 교과서 (항상 펼쳐져 있음) | 주문서 (필요할 때 꺼냄) |
| **함께 쓰면** | CLAUDE.md의 규칙 + 스킬의 지시가 합쳐짐 | |

> 💡 CLAUDE.md = "나는 누구인가" (정체성)
> 스킬 = "지금 이것을 해라" (행동)
> 둘 다 쓰면 최강이에요.
