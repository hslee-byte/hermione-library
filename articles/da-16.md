# DA 수업 #16 — 주문의 경계선 ⚖️

> AI가 편해지면 위험한 순간이 와요.
> "그냥 AI가 한 거니까 맞겠지" — 이 생각이 가장 무서운 주문이에요.

---

## 경고 — AI Holic 자가진단

| 증상 | 단계 |
|---|---|
| AI 결과를 검증 없이 복붙하기 시작 | ⚠️ 주의 |
| "내가 직접 하는 것보다 AI가 낫지"라고 생각 | ⚠️⚠️ 경고 |
| AI 출력에 내 판단을 한 줄도 안 넣음 | 🚨 위험 |
| AI가 틀렸는데 모르고 그대로 제출/발송 | 🚨🚨 사고 |

> 💡 비유: 마법에 의존하다 자기 힘을 잃는 마법사 — 해리 포터에서 톰 리들의 일기장에 빠진 지니 위즐리를 기억하세요. 편리한 도구에 판단을 넘기면, 도구가 당신을 쓰는 거예요.

---

## 핵심 원칙 — AI에게 맡길 것 vs 내가 할 것

| AI에게 맡겨도 되는 것 | 반드시 내가 해야 하는 것 |
|---|---|
| 초안 작성 (Draft) | **최종 판단 (Final Decision)** |
| 정보 수집·정리 | **사실 확인 (Fact-check)** |
| 형식 변환 (표, 요약, 번역) | **맥락 판단 (이 상황에 적절한가)** |
| 패턴 분석·통계 | **윤리적 결정** |
| 후보 리스트 생성 | **최종 선택** |
| 문장 다듬기 | **내 목소리·관점 넣기** |
| 일정·루틴 자동화 | **관계·신뢰 구축 (대면 소통)** |

> 💡 한 줄 요약: **AI는 참모(Staff), 나는 사령관(Commander).** 참모가 보고서를 쓰지만, 서명은 사령관이 해요.

---

## "AI가 틀렸는데 모르고 넘기는 게 진짜 위험"

### 실제로 일어나는 사고 유형

| 사고 유형 | 예시 | 결과 |
|---|---|---|
| 환각(Hallucination) | 존재하지 않는 판례를 인용 | 법정에서 신뢰 상실 |
| 오래된 정보 | 개정 전 세법으로 답변 | 세무 오류 → 가산세 |
| 맥락 오독 | 일반론으로 특수 사례에 답변 | 환자에게 부적절한 안내 |
| 수치 오류 | 계산은 맞는데 단위가 다름 | 재무 보고서 오류 |

> 💡 AI의 환각(Hallucination)이란: AI가 사실이 아닌 내용을 자신 있게 말하는 현상이에요. 마치 기억 조작 주문(Obliviate)을 맞은 사람이 거짓 기억을 진짜라고 확신하는 것처럼요.

## 경계선 체크리스트 — 주문 결과 검증법

| # | 체크 항목 | 확인 |
|---|---|---|
| 1 | 고유명사(인명, 판례번호, 약품명)가 실제로 존재하는가? | ☐ |
| 2 | 숫자·통계의 출처가 명시되어 있는가? | ☐ |
| 3 | 최신 정보인가? (법률·의료·세법은 개정 주기 확인) | ☐ |
| 4 | 내 상황의 맥락이 정확히 반영되었는가? | ☐ |
| 5 | 이 결과를 그대로 써도 내 이름을 걸 수 있는가? | ☐ |

> 5번이 가장 중요해요. **내 이름을 걸 수 없으면, 아직 검증이 안 된 거예요.**

---

## 세계관 정리 — 주문의 3가지 규칙

| 규칙 | 의미 |
|---|---|
| 주문은 후보를 뽑아줄 뿐 | AI 결과는 초안이지 최종본이 아님 |
| 판단은 마법사의 영역 | 최종 결정·서명·책임은 전문가 본인 |
| 검증 없는 주문은 저주(Curse)가 됨 | 확인 안 하면 사고로 이어짐 |

> 에테르스트로 방어 주문(Protego)이 있듯, 전문가에겐 검증이라는 방어 주문이 있어요.
