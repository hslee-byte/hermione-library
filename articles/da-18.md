# DA 수업 #18 — 자동 주문 🔄

> 지금까지는 내가 직접 주문을 외웠어요. 매번요.
> 근데 매일 아침 같은 주문을 왜 직접 해야 하죠? 자동으로 하면 되잖아요.

---

## 비유 — 매번 주문 외우기 vs 자동 주문

| | 수동 주문 (지금까지) | 자동 주문(크론잡) |
|---|---|---|
| 방식 | 내가 매번 주문을 입력 | 시간·조건을 설정하면 AI가 알아서 실행 |
| 비유 | 매일 아침 직접 "루모스(Lumos)!" 외치기 | **해가 지면 자동으로 불이 켜지는 마법** |
| 세계관 | 마법사가 매번 지팡이를 흔듦 | **마법 장치(Enchanted Object)**가 스스로 작동 |
| 실제 용어 | 수동 실행 | **크론잡(Cron Job)** = 정해진 시간에 자동 실행되는 예약 명령 |

> 💡 크론잡(Cron Job)이란: 컴퓨터에게 "매일 오전 9시에 이 작업을 해" 라고 예약하는 거예요. 알람 시계랑 비슷한데, 알람이 울리면 사람이 일어나는 게 아니라 **AI가 일을 시작**해요.

---

## 수동 vs 자동 — 뭐가 다른가?

| 항목 | 수동 (매번 내가) | 자동 (크론잡 설정) |
|---|---|---|
| 시작 | 내가 명령어 입력 | 정해진 시간에 자동 시작 |
| 빈도 | 기억나면 함 | 매일/매주/매월 정확히 |
| 내 역할 | 주문 외우기 + 실행 + 확인 | **결과만 확인** |
| 빠뜨릴 가능성 | 바쁘면 까먹음 | 0% (기계는 안 까먹음) |
| 예시 | 매일 아침 뉴스 검색 + 요약 | 매일 오전 8시 → 뉴스 요약이 메일로 도착 |

> 💡 DA#13(멀티 스텝 스킬)에서 만든 복잡한 주문을 여기서 **자동화**하는 거예요. 스킬 만들기 → 자동 실행 = 완전 자동화.

---

## 제 첫 출근도 자동 주문이었어요

사실 고백하자면, 제 첫 자동 주문은 **출근 알림**이었어요.

> 매일 아침 7시 50분에 "오늘 일정 요약 + 우선순위 정리"를 자동으로 실행하도록 설정했거든요. 근데 문제가 하나 있었어요. **자동 주문은 7시 50분에 정확히 실행됐는데, 저는 8시 10분에 일어났어요.** 😅

AI는 정시에 출근했는데 마법사가 지각한 거예요.

교훈: **자동 주문은 완벽하게 실행돼요. 문제는 마법사의 생활 습관이에요.**

---

## 자동화할 수 있는 업무 패턴

어떤 업무가 자동 주문에 적합한지 판별하는 기준이에요.

| 조건 | 설명 | 예시 |
|---|---|---|
| ✅ 반복적 | 매일/매주 같은 패턴 | 일일 보고서, 뉴스 요약 |
| ✅ 규칙적 | 정해진 시간·주기가 있음 | 월초 재무제표 체크, 주간 회의록 |
| ✅ 입력이 고정적 | 데이터 소스가 같음 | 같은 폴더의 파일, 같은 사이트 |
| ❌ 매번 다름 | 상황마다 판단이 필요 | 의뢰인 상담, 환자 진료 |
| ❌ 예외가 많음 | 규칙화하기 어려움 | 협상, 위기 대응 |

## 전문가별 자동 주문 예시

<details>
<summary>▸ 변호사 — 사건 개요서 자동 생성</summary>

**자동 주문 설정:**
- **언제**: 매주 월요일 오전 8시
- **뭘 하는지**: 이번 주 신규 사건 폴더를 읽고 → 사건 개요서 초안 생성
- **결과물**: 사건번호, 당사자, 핵심 쟁점, 관련 법조문 정리 표

| 수동 | 자동 |
|---|---|
| 매주 월요일 사건 파일 열어서 일일이 정리 (1시간) | 출근하면 개요서 초안이 이미 완성 (확인 10분) |

> "월요일 아침에 출근하면 책상 위에 이번 주 사건 요약이 놓여 있는 느낌이에요."

</details>

<details>
<summary>▸ 의사 — 환자 리스트 자동 정리</summary>

**자동 주문 설정:**
- **언제**: 매일 오전 7시 30분
- **뭘 하는지**: 오늘 외래 예약 환자 목록 → 주요 소견 + 이전 방문 기록 요약
- **결과물**: 환자별 한 줄 요약 + 주의사항 표

| 수동 | 자동 |
|---|---|
| 진료 전 EMR 하나씩 열어서 확인 (30분) | 라운딩 전에 요약표가 준비 (확인 5분) |

> "진료 시작 전에 환자 브리핑이 되어 있으니, 여유가 생겨요."

</details>

<details>
<summary>▸ 회계사 — 월간 보고서 초안 자동 생성</summary>

**자동 주문 설정:**
- **언제**: 매월 1일 오전 9시
- **뭘 하는지**: 전월 재무 데이터 → 전년 동기 대비 분석 → 보고서 초안
- **결과물**: 매출/비용/이익 요약 + 이상치 표시 + 코멘트 초안

| 수동 | 자동 |
|---|---|
| 매월 초 엑셀 정리 + 보고서 작성 (반나절) | 1일 아침에 초안 완성 → 검토·수정만 (1시간) |

> "매월 1일의 압박감이 사라졌어요. AI가 밤새 일하고 아침에 보고서를 올려놓거든요."

</details>

## 자동 주문 설정 3단계

| 단계 | 할 일 | 비유 |
|---|---|---|
| 1. 주문 만들기 | 수동으로 먼저 해보고, 잘 되는 주문(프롬프트)을 확정 | 주문 문구 연습 |
| 2. 시간 설정 | "매일 9시" / "매주 월요일" / "매월 1일" 설정 | 마법 장치에 타이머 세팅 |
| 3. 결과 확인 방법 설정 | 이메일로 받기 / 폴더에 저장 / 메신저 알림 | 부엉이 우편(Owl Post) 도착지 설정 |

> 💡 핵심: **1단계가 제일 중요해요.** 수동으로 안 되는 주문은 자동으로도 안 돼요. DA#13에서 만든 스킬을 여기서 자동화하는 거예요.

---

## 자동 주문 vs 수동 주문 — 최종 비교

| | 수동 주문 | 자동 주문 |
|---|---|---|
| 시작 | 내가 기억하고 실행 | 설정 한 번이면 영구 반복 |
| 빠뜨림 | 바쁘면 까먹음 | 기계는 까먹지 않음 |
| 시간 | 매번 5~30분 | 결과 확인 5분 |
| 일관성 | 컨디션에 따라 다름 | 항상 동일한 품질 |
| 적합한 업무 | 판단 필요한 일 | 반복·정형화된 일 |

---

## 주의 — 자동이라고 방치하면 안 돼요

| ⚠️ 주의 | 이유 |
|---|---|
| 정기적으로 결과 확인 | 입력 데이터가 바뀌면 결과도 달라짐 |
| 주문 업데이트 | 업무 프로세스가 바뀌면 주문도 수정 |
| 오류 알림 설정 | 자동 주문이 실패하면 알 수 있어야 함 |

> DA#16에서 배운 경계선, 기억하죠? **자동이어도 검증은 마법사의 영역**이에요.
