## 뭐가 달라진 거냐면

| | DA#8 (기본편) | DA#20 (심화편) |
|---|---|---|
| 방식 | Claude Code에 엑셀 파일을 넣고 분석 | Claude가 **엑셀 안에** 사이드 패널로 상주 |
| 조작 | 결과를 복사해서 엑셀에 붙여넣기 | **엑셀을 직접 수정** (피벗, 차트, 수식) |
| 이해도 | 파일 단위 분석 | **워크북 전체** (모든 탭, 수식, 종속성) |

> 💡 비유: DA#8은 전화 상담(파일 보내고 답 받기), DA#20은 옆자리에 앉은 동료(엑셀을 직접 조작)예요.

## Claude in Excel이 할 수 있는 것

| 기능 | 주문(프롬프트) 예시 |
|---|---|
| **피벗테이블 조작** | "월별, 카테고리별 매출 합계로 재구성해줘" |
| **조건부 서식** | "매출 100만 미만이면 빨간색으로 표시해줘" |
| **차트 수정** | "Y축 제목을 매출(억원)으로 바꿔줘" |
| **수식 디버깅** | "이 시트의 순환참조를 찾아서 설명해줘" |
| **시나리오 분석** | "원가가 15% 오르면 이익률이 어떻게 변해?" |
| **데이터 검증** | "D열에 드롭다운 목록을 만들어줘" |
| **데이터 바** | "E열(이익률)에 파란색 데이터 바를 넣어줘" |

## 실전 사례 — LBO 모델 디버깅

한 재무 모델러의 경험:

| 항목 | Before (머글 방식) | After (주문) |
|---|---|---|
| 모델 규모 | 10개 시트, 500개+ 수식 | 동일 |
| 순환참조 | 23개 체인, 수작업 추적 | Claude가 30초에 전체 맵핑 |
| 디버깅 시간 | **1주일** | **3시간** |
| 발견 | 못 찾던 버그 | 운전자본(Working Capital) 가정의 숨은 순환참조 발견 |

> "몇 주 동안 못 찾던 걸 Claude가 잡아냈어요." — 어머, 이거 진짜 되는 거예요?! 😅

## 다른 AI와 뭐가 다른가

| | 다른 AI | Claude in Excel |
|---|---|---|
| 엑셀 파일을 | "읽어요" (데이터만) | "**이해해요**" (구조+수식+종속성) |
| 수식 체인 | 셀 값만 봄 | 시트 간 수식 종속성을 **추적** |
| 순환참조 | 에러라고만 표시 | 의도적 vs 실수 **구분** |
| 작업 범위 | 한 시트 | **워크북 전체** (10+ 시트 동시) |

> 💡 비유: 다른 AI는 책 한 페이지를 읽어주는 거고, Claude는 도서관 전체 목록을 외우고 있는 거예요 📖
