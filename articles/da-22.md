# DA 수업 #22 — 호박주스 아끼는 법 🎃

> 오후 2시에 "사용량 한도에 도달했습니다"가 뜬 적 있으세요?
> 호박주스(토큰)를 아끼는 법을 알면, 하루 종일 마법을 쓸 수 있어요.

---

## 먼저 — 왜 호박주스가 빨리 바닥나는 거냐면

Claude는 메시지를 보낼 때마다 **대화 이력 전체를 처음부터 다시 읽어요**.

| 대화 길이 | Claude가 읽는 양 |
|---|---|
| 첫 메시지 | 내 메시지 1개 |
| 10번째 메시지 | 이전 대화 9개 + 내 메시지 1개 |
| 50번째 메시지 | 이전 대화 49개 + 내 메시지 1개 |

> 💡 비유: 매번 교과서 1페이지부터 다시 읽고 나서 답하는 거예요. 대화가 길수록 매번 읽는 양이 늘어나요. 그래서 호박주스(토큰)가 기하급수적으로 소모돼요.

---

## 핵심 절약법 — 효과 큰 순서로

### 1. 파일은 변환해서 올리기

| 형식 | 토큰 비용 | 절약법 |
|---|---|---|
| PDF 1페이지 | 1,500~3,000 토큰 | 텍스트만 뽑아서 마크다운으로 |
| 스크린샷 (1000x1000) | ~1,300 토큰 | 필요한 부분만 크롭 |
| DOCX/PPTX | 보이지 않는 메타데이터 포함 | 텍스트 복사 → 마크다운 |

```
절약 주문:
PDF 대신 → Google Docs에 붙여넣기 → 마크다운으로 다운로드 → 업로드
```

> 이것만 해도 **70% 절약** 가능해요!

---

### 2. 생각은 채팅에서, 만들기는 Cowork에서

| 단계 | 어디서 | 토큰 비용 |
|---|---|---|
| 구조 잡기, 방향 논의 | 채팅(Chat) | 저렴 |
| 파일 생성, 코드 작성 | Cowork | 비쌈 |

```
✅ 좋은 방법: 채팅에서 "보고서 구조를 먼저 잡아줘" → 확정 후 → Cowork에서 "이 구조로 만들어줘"
❌ 나쁜 방법: Cowork에서 바로 "보고서 써줘" → 수정 → 수정 → 수정...
```

> 💡 비유: 설계도를 먼저 그리고(채팅) 건물을 짓는 거예요(Cowork). 설계 없이 바로 짓다가 부수고 다시 짓는 건 비용 폭탄이에요.

---

### 3. "질문부터 해줘" 전략

| 방식 | 토큰 | 결과 |
|---|---|---|
| 500단어 프롬프트 매번 | 매번 500토큰+ | 내가 다 생각해야 함 |
| "질문부터 해줘" 15단어 | 15토큰으로 시작 | AI가 필요한 것만 물어봄 |

```
"이걸 하고 싶어. 내 폴더를 읽고 질문부터 해줘."
```

> 이 15단어로 시작하면, AI가 알아서 필요한 것만 물어봐요. 500단어를 쓸 필요 없어요!

---

### 4. 여러 작업을 한 메시지에 묶기

| 방식 | 대화 이력 재로딩 | 토큰 비용 |
|---|---|---|
| 질문 3개를 따로 | 3번 | 3배 |
| 질문 3개를 하나로 | 1번 | 1배 |

```
❌ 따로 보내기:
"이 기사 요약해줘" → (답변) → "핵심 포인트 뽑아줘" → (답변) → "제목 제안해줘"

✅ 묶어서 보내기:
"이 기사를 1) 3줄 요약 2) 핵심 포인트 5개 3) 제목 3개 제안해줘"
```

> 💡 같은 결과, 토큰 1/3. 게다가 Claude가 전체 그림을 보니까 답변 품질도 올라가요!

---

### 5. "이 부분만 다시" 전략

| 방식 | 토큰 비용 |
|---|---|
| "보고서 전체 다시 써줘" | 전체 재생성 — 비쌈! |
| "3번 섹션만 다시 써줘" | 그 부분만 — 저렴! |

```
"3번 섹션만 다시 써줘. 나머지는 그대로 두고.
설명 없이 결과물만 줘."
```

> "설명 없이 결과물만 줘" — 이 한 마디를 추가하면 불필요한 설명이 사라져서 토큰이 더 절약돼요.

## 추가 절약 팁 8가지

<details>
<summary>▸ 6~13번 펼치기</summary>

| # | 팁 | 설명 |
|---|---|---|
| 6 | 프롬프트 템플릿 재사용 | 같은 구조의 프롬프트는 캐싱 혜택. 30단어 템플릿을 만들어 변수만 교체 |
| 7 | 메시지 수정 기능 쓰기 | 채팅에서 이전 메시지를 Edit → 재생성. 대화가 쌓이지 않음 |
| 8 | 대화 나누기 | 한 세션에서 3시간 대화하지 말고, 주제별로 새 세션. DA#2에서 배운 "전화 끊고 다시 걸기" |
| 9 | 간단한 모델 쓰기 | 간단한 작업은 Haiku(빠르고 저렴), 복잡한 분석만 Opus. DA#11 참고 |
| 10 | 출력 형식 지정 | "200자 이내로" "표로만" — 불필요한 서술 방지 |
| 11 | 불필요한 맥락 제거 | 파일 업로드 전에 필요 없는 부분 삭제 |
| 12 | 여러 버전 동시 요청 금지 | "3가지 버전으로 써줘" → 3배 비용. 하나씩 다듬는 게 저렴 |
| 13 | 음성 입력 활용 | 타이핑보다 말하기가 한 번에 더 풍부한 맥락 제공 → 왕복 횟수 감소 |

</details>

## Before → After

| | Before | After |
|---|---|---|
| 한도 도달 | 매일 오후 2시 | 한 달에 1~2번 |
| 파일 업로드 | PDF 그대로 | 마크다운 변환 (70% 절약) |
| 프롬프트 | 매번 500단어 | "질문부터 해줘" 15단어 |
| 수정 요청 | "전체 다시" | "이 부분만 다시" |
| 질문 방식 | 하나씩 따로 | 한 메시지에 묶기 |
