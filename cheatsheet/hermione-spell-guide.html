🧙 역할 (마법서 / Persona)

"당신은 ~입니다" — AI가 전문가처럼 답변하도록 역할을 부여합니다.

예: "당신은 10년 경력의 수석 마케팅 전략가입니다."

🎯 작업/목표 (주문 / Task)

구체적인 행동과 기대하는 결과물을 명확하게 지시합니다.

예: "인스타 릴스용 15초 광고 스크립트 3개를 작성해주세요."

📋 맥락 (배경 / Context)

왜, 누구를 위해, 어떤 제약조건이 있는지 배경을 설명합니다.

예: "타깃: 20대 대학생 / 예산: 5천만원"

📐 형식/기대치 (결과물 / Format)

표, 목록, 길이, 톤 등 결과물의 형태를 지정합니다.

예: "표 형식으로 정리해주세요."
💡 "It's leviOsa, not levioSA" — 주문 문구가 정확해야 효과가 나요 🪶

🪄 핵심 주문 기법

예시 제공 방식

🎯 Zero-shot (바로 지시)

예시 없이 바로 지시. 간단한 작업에 적합.

예: "다음 문장의 감정을 분류해줘: 오늘 기분이 좋아요"

📝 Few-shot (예시 먼저 보여주기)

1~3개 예시 제공. 형식/스타일 일관성 확보.

예: "영화: 타이타닉 → 💕🚢😭
영화: 어벤져스 → 🦸‍♂️💥🌍
영화: 기생충 → ?"

사고 유도 방식

🔗 Chain-of-Thought (단계별 생각하게 하기)

"단계별로 생각해봐" 또는 "Let's think step by step"

수학, 논리, 복잡한 분석에 효과적

🎭 역할 부여 (전문가처럼 답변하게 하기)

"당신은 20년 차 파이썬 개발자입니다" → 전문 용어와 베스트 프랙티스 반영

⚠️ 이것만은 피하세요!

  • ❌ "~하지 마" 대신 "~해줘" — 부정형보다 긍정형이 정확
  • ❌ 애매한 표현 피하기 — "짧게 써줘" → "3문장 이내로 써줘"
  • ❌ 한 번에 너무 많이 요청 — 복잡한 건 단계별로 나눠서
  • ✅ 결과 보고 반복 다듬기 — "더 짧게", "예시 추가해줘"

⚙️ 고급 설정

  • 🌡️ Temperature
    낮음(0.0~0.3): 분석, 코딩 → 일관된 답변
    높음(0.7~1.0): 창작, 브레인스토밍 → 다양한 답변
  • 🚨 헛소리(환각) 방지
    "모르면 모른다고 해줘" "출처를 인용해줘"

<document>...</document>, <question>...</question> 등으로 내용을 구분하면 AI가 각 부분의 역할을 명확히 파악해요.
